# 视频批量重命名工具 · 使用手册

这是一份面向最终用户的操作手册。即使你不懂编程，也能照着一步步完成视频批量改名。

---

## 一、这个工具能做什么

把一个（或多个）文件夹里的视频，**统一搬到该文件夹的根目录并批量改名**，常用于给短剧、课程等成批视频整理成 `1.mp4 / 2.mp4` 或 `第1集.mp4 / 剧名-第1集.mp4` 这样的规整名字。

- 支持的视频格式：`.mp4 .mov .mkv .avi .wmv .flv .webm .m4v .ts`
- 改名前一定会先「预览」，确认无误才真正执行
- 每次执行都会留日志，**可以一键撤销**
- 不会删除你的非视频文件（图片、字幕、文档等都会原样保留）

下面是软件界面分区示意（① 选择文件夹、② 命名设置、③ 预览清单，中间是操作按钮条与红色提示行）：

## 二、运行前的准备

### 方式 A：直接运行源码（推荐先装 Python）
1. 安装 Python 3.10 或更高版本（[python.org](https://www.python.org/) 下载，安装时勾选「Add Python to PATH」）。
   - 本工具只用 Python 自带的库，**不需要联网装任何第三方包**。
2. 在项目根目录双击 `run.bat`，或在 PowerShell 里运行：`python src\main.py`

### 方式 B：使用打包好的 EXE
到 [Releases](https://github.com/wudiazui/video-batch-renamer/releases) 下载 `video-batch-renamer-vX.Y.Z.exe`，直接双击即可运行，无需安装 Python。

## 三、完整操作流程（4 步）

![四步流程](docs/images/workflow.svg)

### 第 1 步：添加文件夹
- 点击 **「添加文件夹」**，选中要处理的视频文件夹。
- 可以多次点击，**添加多个文件夹**，它们会出现在下面的列表里，并显示每个文件夹有多少个视频。
- 选错了？选中列表里的项，点 **「移除选中」**（或按 `Delete` 键）；想全清掉点 **「清空列表」**。
- 小技巧：双击列表里的文件夹可在资源管理器中打开它；右键有快捷菜单。

### 第 2 步：选择命名设置
有两种模式，二选一：

**模式一 · 连续编号**
- 适合文件名杂乱、只想按顺序重新编号的情况。
- 填「起始数字」，例如填 `1`，就会得到 `1.mp4、2.mp4、3.mp4……`
- 多个文件夹时，如果勾选 **「跨文件夹连续编号」**，编号会接续下去（第一个文件夹 1、2、3，第二个文件夹接着 4、5……）。不勾选则每个文件夹都从起始数字重新开始。

**模式二 · 识别集数**
- 适合文件名里本来就有集数的情况。除了 `第1集`、`01`、`第十一集`，像 `超清-6`、`EP6`、`S01E06`、`1080P-6` 这种**带数字但没有「集」字**的也能认出来。
- 会自动忽略清晰度/编码/帧率等噪声（`1080P`、`2k`、`4K`、`x264`、`1920x1080`、`60帧`、`v2` 等），不会把它们当成集数；纯清晰度名和四位年份也不会误判。
- 可填「剧名」，配合模板 `{title}-第{episode}集` 输出成 `庆余年-第1集.mp4`。

**通用设置**
- 补零：`1`（不补零）、`2`（→ 01）、`3`（→ 001）。
- 模板：见下方对照表。
- 保持扩展名大小写：默认会把扩展名统一成小写（`.MP4` → `.mp4`），勾上则保留原样。

### 第 3 步：生成预览
- 点 **「生成预览」**，下方表格会列出每个文件「原文件名 → 新文件名」。
- **绿色**＝没问题，**红色**＝有错误。出现红色时，下方会有一行红字提示该怎么解决（见第五节）。
- 只要还有红色错误，「确认执行改名」按钮就点不动。
- 小技巧：**双击预览里的任意一行**，可在资源管理器中定位到对应文件，方便先核对来源。
- 小技巧：识别集数模式下，**在预览某一行上右键 →「手动设置集数」**，填入第几集即可——自动识别失败的红行补上集数就能改名，识别对的也能手动覆盖，改完会即时重算。
- 注意：生成预览后又改了任何设置，预览会自动失效，需要**重新点「生成预览」**。

### 第 4 步：确认执行
- 确认预览全绿后，点 **「确认执行改名」**，再次确认弹窗即可。
- 执行时下方有进度条；完成后会提示成功改了多少个、删了多少个空文件夹。

## 四、命名模板对照表

| 模板 | 适用模式 | 输入示例 | 输出示例 |
|------|----------|----------|----------|
| `{number}` | 连续编号 | 起始数字 1 | `1.mp4`、`2.mp4` |
| `EP{number}` | 连续编号 | 起始数字 1、补零 2 | `EP01.mp4`、`EP02.mp4` |
| `clip-{number}` | 连续编号 | 起始数字 1 | `clip-1.mp4` |
| `第{episode}集` | 识别集数 | 文件名含 第1集 | `第1集.mp4` |
| `{title}-第{episode}集` | 识别集数 | 剧名「庆余年」 | `庆余年-第1集.mp4` |
| `EP{episode}` | 识别集数 | 文件名含 01 | `EP01.mp4` |

> `{number}` 是连续编号，`{episode}` 是从文件名里识别出来的集数，`{title}` 是你填的剧名。

## 五、预览里出现红色错误怎么办

| 红色状态 | 含义 | 解决办法 |
|----------|------|----------|
| 无法识别集数 | 「识别集数」模式下，某文件名里找不到集数 | **在预览里右键该行「手动设置集数」直接补上第几集**；或改用「连续编号」模式 |
| 目标文件名重复 | 多个文件会被改成同一个名字 | 检查不同文件夹里是否有相同的集数/编号 |
| 目标已存在 | 目标位置已经有同名文件 | 换个起始数字或剧名，或先处理掉那个同名文件 |
| 目标文件名包含非法字符 | 剧名/模板里有 `\ / : * ? " < > \|` | 删掉这些字符 |
| 目标文件名包含首尾空格 | 剧名/模板开头或结尾有空格 | 去掉首尾空格 |
| 目标文件名为系统保留名 | 名字是 Windows 保留名（CON、PRN、NUL 等） | 改剧名或模板，避开这些词 |
| 配置错误 | 命名设置本身有问题 | 检查起始数字、剧名、模板是否填对 |

## 六、撤销（改错了不要慌）

工具会在**每个被处理的文件夹**里生成 `_rename_logs\rename_log_日期-时间.csv` 日志。

- **撤销最近一次**：自动找到列表内所有文件夹中**时间最新的**那条日志并回滚。
- **选择日志撤销**：手动挑一个具体的 CSV 日志来回滚（适合撤销较早的某一次）。

撤销同样会先检查：如果原位置已经被新文件占用，会拒绝撤销并提示，不会覆盖你的文件。

## 七、常见问题 FAQ

**Q：会把多个文件夹的视频合并到一个文件夹吗？**
A：不会。即使用「跨文件夹连续编号」，文件也只是各自在自己的文件夹内被改名，不会跨文件夹搬动。

**Q：为什么一定要先生成预览？**
A：批量改名是高影响操作，预览让你在真正动手前看清每个文件改成什么、有没有冲突。

**Q：我的字幕、封面图会被动到吗？**
A：不会。工具只处理上面列出的视频格式，其它文件原样保留；只有「移走视频后变空」的子文件夹才会被清理。

**Q：处理到一半出错了，文件会不会损坏？**
A：不会。执行采用「先暂存再写入」并带回滚；任何一步失败都会尽量把文件恢复原状，并在日志里记录。

**Q：能处理网络盘 / 移动硬盘上的视频吗？**
A：可以，只要该路径在资源管理器里能正常访问即可。

**Q：上次用过的文件夹和设置会记住吗？**
A：会。文件夹列表、命名设置、窗口大小都会保存在 `%APPDATA%\VideoRenamerGUI\settings.json`，下次打开自动恢复。

**Q：只有几个文件识别不出集数，要全部改用连续编号吗？**
A：不用。在预览里**右键那几行红的 →「手动设置集数」**，填上第几集即可，其余文件仍按自动识别的结果改名。

## 八、安全建议

第一次处理重要素材前，**强烈建议先复制一份小样本测试**。预览能挡掉大部分问题，但批量改名仍是不可轻视的操作。
